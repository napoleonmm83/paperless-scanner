# F-007 — BillingManager state machine missing — null-safety + init race risks

**Severity:** `high` · **Area:** `security/billing` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/34

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/billing/BillingManager.kt`
- **Lines:** `L68-L100, L258`
- **Target:** `BillingManager.initialize, BillingManager.launchPurchaseFlow`

## Problem

`launchPurchaseFlow()` accesses `billingClient?.launchBillingFlow` without verifying init succeeded. `queryProducts/Purchases` only check `isReady`. If init fails silently the app sits in a half-broken state. Commit cc0b331 fixed crashes via activity-state checks but the underlying state machine is still implicit.

## Proposed Fix

Introduce explicit `BillingState` enum (UNINITIALIZED/INITIALIZING/READY/FAILED). `initialize()` retries with backoff or transitions to FAILED. All public methods fail fast if not READY and emit an analytics event. Add unit tests for init failure / race scenarios.

## Acceptance Criteria

- [ ] BillingState enum + transitions encapsulated in BillingManager
- [ ] launchPurchaseFlow rejects when not READY (no NPE risk)
- [ ] Init failure path tested (no leaked listeners, idempotent retry)
- [ ] Tests still pass
