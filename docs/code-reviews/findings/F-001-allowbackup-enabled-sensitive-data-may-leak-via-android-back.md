# F-001 — allowBackup enabled — sensitive data may leak via Android Backup

**Severity:** `high` · **Area:** `security/manifest` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/28

## Location

- **File:** `app/src/main/AndroidManifest.xml`
- **Lines:** `L24`

## Problem

The manifest sets `android:allowBackup="true"` globally. This allows backup/restore of sensitive data (auth tokens, AppLock hash, server URL) via adb/recovery. While EncryptedSharedPreferences provides encryption, the Android backup mechanism can extract data during device-compromise scenarios.

## Proposed Fix

Set `android:allowBackup="false"` in AndroidManifest. If legitimate backup of UI prefs is desired, implement a `BackupAgent` that explicitly excludes sensitive SharedPreferences/DataStore files. See Android Security Guide / Data Storage / Backup.

## Acceptance Criteria

- [ ] android:allowBackup is false OR a BackupAgent excludes encrypted/secure storage
- [ ] No sensitive files appear in `bmgr backupnow` output for this app
