# F-014 — Sync TokenManager APIs lack @WorkerThread — IDE can't warn against main-thread misuse

**Severity:** `low` · **Area:** `security/logging` · **Effort:** `XS`

- **File:** `app/src/main/java/com/paperless/scanner/data/datastore/TokenManager.kt`
- **Lines:** `L609-L734`
- **Target:** `TokenManager.*Sync`

## Problem

All `*Sync()` blocking helpers lack `@WorkerThread` annotations, so the IDE/lint cannot flag accidental main-thread use, increasing ANR risk for future contributors.

## Proposed Fix

Add `@WorkerThread` from `androidx.annotation` to every blocking API and document blocking semantics in kdoc.

- [ ] @WorkerThread annotation on all sync APIs
- [ ] Lint passes; no main-thread call sites flagged
