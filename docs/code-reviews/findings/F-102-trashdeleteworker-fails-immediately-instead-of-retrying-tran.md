# F-102 — TrashDeleteWorker fails immediately instead of retrying transient errors

**Severity:** `medium` · **Area:** `worker/retry` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/129

## Location

- **File:** `app/src/main/java/com/paperless/scanner/worker/TrashDeleteWorker.kt`
- **Lines:** `L122-L124`
- **Target:** `TrashDeleteWorker.onFailure`

## Problem

Any failure leads to `Result.failure()` and the soft-deleted document never gets retried. Transient 5xx / network errors lose data permanently from the trash queue.

## Proposed Fix

Inspect error type: 5xx + IOException → `Result.retry()` with backoff (`setBackoffCriteria(BackoffPolicy.EXPONENTIAL, 15, MINUTES)`). 4xx auth → `Result.failure()` with a notification CTA.

## Acceptance Criteria

- [ ] Transient errors retry
- [ ] Auth errors surface to UI
- [ ] setBackoffCriteria configured
- [ ] Tests cover both branches
