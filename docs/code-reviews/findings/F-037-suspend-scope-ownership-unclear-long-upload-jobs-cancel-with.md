# F-037 — Suspend scope ownership unclear — long upload jobs cancel with VM

**Severity:** `medium` · **Area:** `architecture/threading` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/63

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/DocumentRepository.kt`
- **Lines:** `L73, L153, L441, L488 (and many more)`
- **Target:** `DocumentRepository (all suspend functions)`

## Problem

Suspend functions inherit caller scope. Upload from `viewModelScope` cancels when the user navigates away. Long-running operations (multi-page upload, batch restore) silently abort with no resume strategy.

## Proposed Fix

Document scope ownership per public method (kdoc). For long-running mutations move to a WorkManager job or `applicationScope`. Persist intent before suspending so reconnect can resume.

## Acceptance Criteria

- [ ] Scope ownership documented per method
- [ ] Long uploads survive screen navigation (Worker or applicationScope)
- [ ] Pending changes flushed on reconnect even after VM cancellation
- [ ] Tests verify cancellation behavior
