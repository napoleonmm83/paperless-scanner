# F-012 — AppLock state can become inconsistent on logout crash

**Severity:** `medium` · **Area:** `security/applock` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/38

## Location

- **File:** `app/src/main/java/com/paperless/scanner/util/AppLockManager.kt`
- **Lines:** `L512-L522`
- **Target:** `AppLockManager.handleFailedAttempt`

## Problem

When MAX_TOTAL_ATTEMPTS is reached, the code clears credentials + AppLock settings in DataStore but only clears `failedAttempts/lockoutUntil` in-memory. A crash between the two leaves inconsistent state: on next launch, no credentials but `LockedOut(permanent=true)` may be reconstructed from stale flags.

## Proposed Fix

Wrap logout in a single atomic flow that clears DataStore-side state AND in-memory counters in one call. Add an init-time invariant check: if no credentials exist, force Unlocked state and log if a stale LockedOut was found.

## Acceptance Criteria

- [ ] Single atomic logout clears all relevant state
- [ ] Init-time invariant detects + repairs inconsistency
- [ ] Crash-during-logout test passes
