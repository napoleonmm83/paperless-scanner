# F-060 — Inline Color literals bypass theme tokens (ScanScreen)

**Severity:** `high` · **Area:** `ui/style-guide` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/88

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/scan/ScanScreen.kt`
- **Lines:** `L546, L556`
- **Target:** `ScanScreen`

## Problem

Hardcoded `Color(0xFF8DD7FF)` and `Color(0xFFB88DFF)` instead of theme tokens. Combined with `contentColor = Color.Black` this assumes a light background and breaks dark mode. CLAUDE.md mandates Dark Tech Precision Pro: Primary `#E1FF8D`, Surface `#141414`, Background `#0A0A0A`.

## Proposed Fix

Replace inline colors with `MaterialTheme.colorScheme.*` or a typed semantic palette (e.g. `LocalScanCategoryColors`). Verify WCAG AA contrast in both light + dark modes.

## Acceptance Criteria

- [ ] No `Color(0x...)` literals in ScanScreen
- [ ] Colors resolved from MaterialTheme or design-system palette
- [ ] Dark Tech Precision Pro tokens used
- [ ] Renders correctly in light + dark mode
