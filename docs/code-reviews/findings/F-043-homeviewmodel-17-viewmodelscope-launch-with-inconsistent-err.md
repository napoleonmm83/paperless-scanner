# F-043 — HomeViewModel: 17+ viewModelScope.launch with inconsistent error handling

**Severity:** `high` · **Area:** `architecture/reactive` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/85

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/home/HomeViewModel.kt`
- **Lines:** `L255-L1264`
- **Target:** `HomeViewModel.observeTagsReactively, observeRecentDocumentsReactively, observeProcessingTasksReactively, etc.`

## Problem

~15 separate launch{collect{}} blocks each handle errors differently — log/swallow/null-update. One repository throwing = silent failure or inconsistent UI state.

## Proposed Fix

Wrap repo flows with a shared helper that maps to `Result<T>` (e.g. `Flow<T>.asUiResult()`). Centralize error UI state into a single `_errorState: StateFlow<HomeError?>`. Standardize onError handling.

## Acceptance Criteria

- [ ] All observe* flows funnel into a shared Result<T> helper
- [ ] Single error StateFlow owns user-visible errors
- [ ] Tests verify error propagation + recovery
