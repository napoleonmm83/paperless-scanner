# F-095 — Write timeout 60s too tight for large multi-page uploads / Cloudflare paths

**Severity:** `high` · **Area:** `performance/networking` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/122

## Location

- **File:** `app/src/main/java/com/paperless/scanner/util/NetworkConfig.kt`
- **Lines:** `L22`
- **Target:** `WRITE_TIMEOUT_SECONDS`

## Problem

60-second write timeout fails large multi-page PDF uploads on slow links. CloudflareDetectionInterceptor warns about the 100s edge limit but no timeout adapts dynamically.

## Proposed Fix

Adaptive write timeout: base 120s + 1–2s per MB. When Cloudflare is detected, cap at 90s and surface a UX warning. Persist last detected outcome in DataStore to avoid re-probing every request.

## Acceptance Criteria

- [ ] Timeout scales with file size
- [ ] Cloudflare-aware cap applied
- [ ] Logs record chosen timeout per request
- [ ] Small uploads see no regression
