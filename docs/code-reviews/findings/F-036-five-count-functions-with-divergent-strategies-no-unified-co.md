# F-036 — Five count functions with divergent strategies — no unified count API

**Severity:** `medium` · **Area:** `data/caching` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/62

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/DocumentRepository.kt`

- **Target:** `DocumentRepository.observeCountWithFilter, observeUntaggedDocumentsCount, getDocumentCount, getUntaggedCount, observeTrashCount`

## Problem

Five count functions each take different inputs, sometimes call DAO, sometimes raw query builder, sometimes accept `forceRefresh`. Caching strategy differs per function.

## Proposed Fix

Create `DocumentCountRepository` exposing a single `observeCount(filter, searchQuery, forceRefresh): Flow<Int>` with thin wrappers for trash/untagged. Push query construction down to DAO.

## Acceptance Criteria

- [ ] DocumentCountRepository owns all counts
- [ ] Single primary method + thin wrappers
- [ ] Cache strategy uniform across counts
