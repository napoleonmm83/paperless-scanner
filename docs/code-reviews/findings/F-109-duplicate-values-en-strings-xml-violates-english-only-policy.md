# F-109 — Duplicate values-en/strings.xml violates English-only policy

**Severity:** `medium` · **Area:** `ui/i18n` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/136

## Location

- **File:** `app/src/main/res/values-en/strings.xml`
- **Lines:** `—`

## Problem

Both `values/strings.xml` (5444 bytes) and `values-en/strings.xml` (81427 bytes) exist. CLAUDE.md mandates a single source: English-only `values/strings.xml`, with Gemini auto-translating 15+ languages (incl. German) on Play Console upload. The duplicate `values-en/` directory was supposed to be removed and would shadow the auto-translation pipeline.

## Proposed Fix

Delete `app/src/main/res/values-en/strings.xml`. Verify all string keys present in `values-en/` are also present in `values/strings.xml`; merge any unique keys. Confirm Gemini still produces English output (it is the source — should be unchanged). Add a CI check to fail builds if any `values-en/` or `values-de/` directory is reintroduced.

## Acceptance Criteria

- [ ] values-en/strings.xml removed
- [ ] All keys consolidated to values/strings.xml
- [ ] CI check rejects re-introducing values-en/ or values-de/ directories
- [ ] Play Console upload still produces 15+ translations
