# F-059 — LabelsViewModel.init triggers refresh() synchronously without error capture

**Severity:** `medium` · **Area:** `architecture/reactive` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/87

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/labels/LabelsViewModel.kt`
- **Lines:** `L123`
- **Target:** `LabelsViewModel.<init>`

## Problem

`init { refresh() }` runs at construction. Any exception thrown propagates from constructor; observers never see an error state.

## Proposed Fix

Move refresh into an observer-first path: have refreshing be a side effect of starting reactive flows, OR `init { viewModelScope.launch { refresh() } }` wrapped in try/catch that updates `_uiState` with a failure.

## Acceptance Criteria

- [ ] VM construction no longer throws on refresh error
- [ ] Failure surfaces as UI error state
- [ ] Tests verify error path
