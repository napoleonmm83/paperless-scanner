# F-084 — Widget deep-link intents bypass AppLock interception

**Severity:** `high` · **Area:** `widget/deep-link` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/111

## Location

- **File:** `app/src/main/java/com/paperless/scanner/widget/ScannerWidget.kt`
- **Lines:** `L732-L745`
- **Target:** `ScannerWidget.createDeepLinkIntent`

## Problem

Intents use explicit `ComponentName(context, MainActivity::class.java)` to avoid Glance trampoline crashes, but this short-circuits the implicit-deep-link resolution that AppLockNavigationInterceptor relies on. Widget actions therefore reach scan/upload UIs without authentication.

## Proposed Fix

Either (a) add an explicit AppLock check in MainActivity for deep-link entries originating from widgets, or (b) route through an internal DeepLink dispatcher that consults AppLockManager before forwarding. Document the trade-off vs trampoline crash risk.

## Acceptance Criteria

- [ ] Widget actions respect AppLock state
- [ ] Trampoline-crash regression on OnePlus/Samsung does not return
- [ ] Manual test: AppLock locked, tap widget → unlock prompt shown
