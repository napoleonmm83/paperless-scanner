# F-106 — UploadWorker may exceed 10-min foreground worker limit on slow links

**Severity:** `critical` · **Area:** `worker/workmanager` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/133

## Location

- **File:** `app/src/main/java/com/paperless/scanner/worker/UploadWorker.kt`
- **Lines:** `L51-L87`
- **Target:** `UploadWorker.doWork`

## Problem

Worker calls `setForeground` once at start. If the upload exceeds ~10 minutes (slow connection or large multi-page PDF) the system can kill it. No periodic ForegroundInfo update or foreground service fallback.

## Proposed Fix

Periodically refresh `setForegroundAsync(...)` (e.g. every 30s) with updated progress. For very large uploads consider a long-running ForegroundService with WorkManager scheduling. Surface a UX warning on uploads >5 minutes.

## Acceptance Criteria

- [ ] ForegroundInfo refreshed periodically
- [ ] 15-minute upload completes without being killed
- [ ] Long-upload UX warning visible
- [ ] Tests verify long-running survival
