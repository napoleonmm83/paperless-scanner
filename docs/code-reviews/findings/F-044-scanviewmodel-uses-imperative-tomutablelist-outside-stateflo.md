# F-044 — ScanViewModel uses imperative .toMutableList() outside StateFlow.update

**Severity:** `high` · **Area:** `architecture/state` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/69

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/scan/ScanViewModel.kt`
- **Lines:** `L544-L577`
- **Target:** `ScanViewModel.undoRemovePage, ScanViewModel.movePage`

## Problem

Page mutations create intermediate `MutableList` outside the StateFlow update block, then sync to SavedStateHandle separately. Forgetting the sync silently loses pages on Process Death/AppLock.

## Proposed Fix

Move all mutations inside `_uiState.update { state -> state.copy(pages = ...) }`. Consolidate SavedStateHandle sync into a helper invoked from a single update path.

## Acceptance Criteria

- [ ] No .toMutableList outside .update {}
- [ ] Single helper updates state + syncs SavedStateHandle atomically
- [ ] Tests verify atomic mutation
