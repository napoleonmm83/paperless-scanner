# F-022 — AuthRepository builds raw OkHttp requests — bypasses Retrofit layer

**Severity:** `medium` · **Area:** `data/api` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/48

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/AuthRepository.kt`
- **Lines:** `L65-L80, L300+`
- **Target:** `AuthRepository.<init>, AuthRepository.detectServerProtocol`

## Problem

AuthRepository injects OkHttpClient directly to do dynamic protocol detection (raw `okhttp3.Request`). This creates a parallel HTTP layer with its own timeout/retry, inconsistent with the Retrofit-based API surface.

## Proposed Fix

Extract a `ProtocolDetector` service that owns the raw HTTP probe with explicit short timeout. AuthRepository becomes a façade: detection → server URL set → Retrofit calls. Centralize timeout configuration.

## Acceptance Criteria

- [ ] ProtocolDetector class with single responsibility
- [ ] AuthRepository no longer references OkHttpClient directly
- [ ] Probe timeouts documented + tested
