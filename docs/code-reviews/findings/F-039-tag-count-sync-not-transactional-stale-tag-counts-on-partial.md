# F-039 — Tag-count sync not transactional — stale tag counts on partial failure

**Severity:** `medium` · **Area:** `data/caching` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/65

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/DocumentRepository.kt`
- **Lines:** `L837-L842, L898-L918`
- **Target:** `DocumentRepository.updateTagDocumentCounts, getOldTagIds`

## Problem

`updateTagDocumentCounts()` mutates tag counts without a Room transaction wrapping the document update. Partial DAO failure leaves counts diverged. `getOldTagIds()` returns `emptyList()` on JSON parse failure, silently losing history.

## Proposed Fix

Wrap document + tag-count mutations in a single `@Transaction` Room method. Log + propagate JSON parse failures rather than swallowing them. Optionally extract `TagSyncService` for clarity.

## Acceptance Criteria

- [ ] Document + tag-count update happens in single Room transaction
- [ ] JSON parse failures logged + surfaced (no silent emptyList)
- [ ] Test verifies atomic rollback on tag DAO failure
