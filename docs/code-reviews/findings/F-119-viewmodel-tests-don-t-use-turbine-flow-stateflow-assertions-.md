# F-119 — ViewModel tests don't use Turbine — Flow/StateFlow assertions implicit

**Severity:** `medium` · **Area:** `testing/test-coverage` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/146

## Location

- **File:** `app/src/test/java/com/paperless/scanner/ui/screens/upload/UploadViewModelTest.kt`
- **Lines:** `—`
- **Target:** `ViewModel tests in general`

## Problem

ViewModel tests rely on `runTest { … advanceUntilIdle() }` then assert the latest `uiState.value`. Intermediate emissions are not captured, so a Flow that emits in the wrong order can still pass.

## Proposed Fix

Add `app.cash.turbine` to test dependencies. Convert StateFlow assertions to `viewModel.uiState.test { assertThat(awaitItem()).isEqualTo(...) }`. Document the pattern in `docs/TECHNICAL.md` test section.

## Acceptance Criteria

- [ ] Turbine added as test dependency
- [ ] All ViewModel tests use Turbine for Flow assertions
- [ ] Intermediate state changes verified
- [ ] Pattern documented for new tests
