# F-027 — Query/paging/filter logic inlined and duplicated across 5 functions

**Severity:** `high` · **Area:** `data/srp` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/53

## Location

- **Lines:** `L395-L606`
- **Target:** `DocumentRepository.observeDocuments, getDocumentsPaged, getDocuments, searchDocuments, observeCountWithFilter`

## Problem

Five list/count functions duplicate paging math + filter joining + cache fallback inline. `DocumentFilterQueryBuilder` is invoked per-call with manual tag-string building (L523).

## Proposed Fix

Create `DocumentListRepository` with unified `observeDocuments(filter, page, pageSize): Flow<PagingData<Document>>`. Move tag filtering into a `TagFilterStrategy`. Push raw query building into the DAO layer.

## Acceptance Criteria

- [ ] Five functions consolidated behind a DocumentListRepository façade
- [ ] No raw tag-string concatenation in repository
- [ ] Paging size parameter is documented + injectable
