# F-085 — WidgetDeviceChecker OnePlus rule too narrow (Android 11 only)

**Severity:** `medium` · **Area:** `widget/legacy` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/112

- **Lines:** `L35-L39`
- **Target:** `WidgetDeviceChecker.isProblematicOnePlus`

## Problem

Rule matches only `Build.VERSION_CODES.R` (API 30). OxygenOS trampoline issues persist on Android 12/13 (OnePlus 10/11/12).

## Proposed Fix

Broaden the match to API 30–33 OR drive the decision from runtime crash telemetry: catch InvisibleActionTrampolineActivity once, set a `force_legacy_widget` flag in SharedPreferences, and use legacy thereafter.

## Acceptance Criteria

- [ ] OnePlus Android 12/13 fall back to legacy widget
- [ ] Crashlytics shows reduced trampoline crash rate
- [ ] False-positive rate on other OEMs unchanged
