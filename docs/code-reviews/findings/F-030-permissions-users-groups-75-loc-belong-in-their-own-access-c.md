# F-030 — Permissions/users/groups (~75 LOC) belong in their own access-control repository

**Severity:** `medium` · **Area:** `refactor/srp` · **Effort:** `S`

- **Lines:** `L968-L1040`
- **Target:** `DocumentRepository.getUsers, getGroups, updateDocumentPermissions`

## Problem

User/group/permission methods are bundled into DocumentRepository for convenience but represent a different domain — access control. They are pure API pass-throughs without document logic.

## Proposed Fix

Extract `PermissionRepository` with these three methods. Document mutations import it for permission resolution; dependency direction Document→Permission only.

## Acceptance Criteria

- [ ] PermissionRepository owns user/group/permission methods
- [ ] DocumentMetadataRepository depends on PermissionRepository, not vice versa
- [ ] Tests for both repositories independent
