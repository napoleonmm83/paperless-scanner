# F-091 — AppWidgetId cast failure path silently falls back to QUICK_SCAN

**Severity:** `medium` · **Area:** `widget/glance` · **Effort:** `S`

- **File:** `app/src/main/java/com/paperless/scanner/widget/ScannerWidget.kt`

- **Target:** `ScannerWidget.provideGlance`

## Problem

If `id as? AppWidgetId` returns null (e.g. test GlanceId), config sync is skipped and the widget defaults to QUICK_SCAN with no log/telemetry.

## Proposed Fix

Log a WARN with the actual GlanceId class on cast failure. Emit an analytics event so unexpected GlanceId types are visible. Document expected types.
