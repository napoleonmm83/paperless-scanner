# F-097 — DynamicBaseUrlInterceptor uses runBlocking on every request

**Severity:** `high` · **Area:** `architecture/interceptor` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/124

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/api/DynamicBaseUrlInterceptor.kt`
- **Lines:** `L29-L31`
- **Target:** `DynamicBaseUrlInterceptor.intercept`

## Problem

Every request blocks reading server URL from DataStore via `runBlocking`. Comment claims OkHttp threading makes it safe, but this still serializes requests under DataStore lock contention.

## Proposed Fix

Pre-resolve the server URL during DI graph construction (or first repository init) into a thread-safe `AtomicReference<String?>`. Update on logout/login. Interceptor reads from the atomic, no blocking. See F-006 for the related TokenManager pattern.

## Acceptance Criteria

- [ ] Interceptor reads from non-blocking holder
- [ ] Holder updated on auth/state transitions
- [ ] No runBlocking in hot path
- [ ] Tests cover URL change scenarios
