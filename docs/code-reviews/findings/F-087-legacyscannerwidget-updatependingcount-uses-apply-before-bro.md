# F-087 — LegacyScannerWidget.updatePendingCount uses .apply() before broadcast

**Severity:** `medium` · **Area:** `widget/legacy` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/114

## Location

- **File:** `app/src/main/java/com/paperless/scanner/widget/LegacyScannerWidget.kt`
- **Lines:** `L40-L52`
- **Target:** `LegacyScannerWidget.updatePendingCount`

## Problem

Saves pending count with `.apply()` then broadcasts an update. Receiver may render before the async write commits → stale count.

## Proposed Fix

Use `.commit()` for the count write; then broadcast. Or move the broadcast into a callback after a successful commit.

## Acceptance Criteria

- [ ] Count synced before broadcast
- [ ] Legacy widget shows accurate value immediately
- [ ] Test simulates rapid updates
