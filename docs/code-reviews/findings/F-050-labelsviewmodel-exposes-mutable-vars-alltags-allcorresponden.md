# F-050 — LabelsViewModel exposes mutable vars (allTags, allCorrespondents, …) with race risk

**Severity:** `high` · **Area:** `architecture/state` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/74

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/labels/LabelsViewModel.kt`
- **Lines:** `L105-L108`
- **Target:** `LabelsViewModel.allTags / allCorrespondents / allDocumentTypes / allCustomFields`

## Problem

Four `private var` lists are updated inside reactive flows but read by other methods (`getActiveEntities`, `applyCurrentFilters`). Read-during-write races; late subscribers don't replay last value.

## Proposed Fix

Convert each to `MutableStateFlow<List<EntityItem>>`. All mutations go through `.update {}`. Expose a derived `displayedEntities: StateFlow<List<EntityItem>>` that filters/applies search reactively.

## Acceptance Criteria

- [ ] Four mutable vars replaced with StateFlow
- [ ] displayedEntities is derived via combine() with no manual triggers
- [ ] Tests verify reactive filter updates
