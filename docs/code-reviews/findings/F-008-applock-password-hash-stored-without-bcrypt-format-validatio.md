# F-008 — AppLock password hash stored without BCrypt format validation

**Severity:** `medium` · **Area:** `security/applock` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/35

## Location

- **File:** `app/src/main/java/com/paperless/scanner/util/AppLockManager.kt`
- **Lines:** `L185-L197`
- **Target:** `AppLockManager.setupAppLock`

## Problem

BCrypt hash is stored without sanity checking that the value persists in `$2a$/$2b$/$2y$` format. If DataStore is corrupted or replaced via backup, `BCrypt.checkpw()` silently returns false and the user is locked out without any indication of corruption vs. wrong password.

## Proposed Fix

Validate BCrypt prefix on write and on read. On format mismatch, log a WARN audit event and force re-setup. Add a migration check at startup. Reference OWASP Password Storage Cheat Sheet.

## Acceptance Criteria

- [ ] Format validation present on setup + unlock paths
- [ ] Hash corruption detected → user re-setup flow
- [ ] Tests verify good / bad hash behavior
