# F-082 — Widget config save: SharedPreferences + Glance state update not atomic

**Severity:** `high` · **Area:** `widget/glance` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/109

## Location

- **File:** `app/src/main/java/com/paperless/scanner/widget/WidgetConfigActivity.kt`
- **Lines:** `L103-L128`
- **Target:** `WidgetConfigActivity.saveConfigAndFinish`

## Problem

Config is written to SharedPreferences, then Glance state is updated in a separate coroutine. A racing `provideGlance` can read the SharedPrefs value but miss the Glance state update, briefly flashing the wrong widget type.

## Proposed Fix

Wrap both writes in a single transactional flow: confirm `editor.commit()` returns true, then `updateAppWidgetState { state -> ... }` synchronously, then call `update()` once. Keep the broadcast as a defensive fallback only.

## Acceptance Criteria

- [ ] SharedPreferences commit verified before Glance state write
- [ ] Glance state + SharedPrefs always agree at render time
- [ ] Tests cover save-then-render race
