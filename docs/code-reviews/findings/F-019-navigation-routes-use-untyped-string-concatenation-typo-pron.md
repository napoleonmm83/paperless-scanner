# F-019 — Navigation routes use untyped string concatenation — typo-prone

**Severity:** `medium` · **Area:** `architecture/navigation` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/45

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/navigation/Screen.kt`
- **Lines:** `L22-L55, L75`
- **Target:** `Screen.Scan.createRoute and call sites`

## Problem

Routes like `"scan?pageUris={pageUris}"` are strings; missing URL encoding silently fails, typos compile. Hardcoded literal strings appear at navigate() call sites.

## Proposed Fix

Adopt typed Navigation 2.8+ with `@Serializable` route data classes, OR centralize route construction via factory functions on each `Screen` and forbid raw string literals at call sites (lint rule).

## Acceptance Criteria

- [ ] All navigate() calls use typed/factory routes — no raw strings in screens
- [ ] URL encode/decode centralized + tested
- [ ] Migration plan documented for any deferred routes
