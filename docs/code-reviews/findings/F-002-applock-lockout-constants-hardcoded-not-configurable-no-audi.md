# F-002 — AppLock lockout constants hardcoded — not configurable, no audit log

**Severity:** `medium` · **Area:** `security/applock` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/29

## Location

- **File:** `app/src/main/java/com/paperless/scanner/util/AppLockManager.kt`
- **Lines:** `L108-L112`
- **Target:** `AppLockManager.Companion`

## Problem

Security-critical brute-force constants (MAX_FAILED_ATTEMPTS=5, LOCKOUT_DURATION_MILLIS=1_800_000, MAX_TOTAL_ATTEMPTS=15) are hardcoded private constants. They cannot be tuned without recompile, and lockout events are never logged for security audits.

## Proposed Fix

Move constants to TokenManager-backed DataStore prefs with sensible defaults, OR document them explicitly. Emit an INFO-level log (no PII / hash) when a lockout threshold is reached so security audits have a trail.

## Acceptance Criteria

- [ ] Constants documented or moved to DataStore with defaults
- [ ] Lockout-threshold reached emits an audit log without sensitive data
- [ ] Tests still pass
