# F-104 — OkHttp client has no Cache configured — list endpoints always hit network

**Severity:** `medium` · **Area:** `performance/caching` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/131

## Location

- **File:** `app/src/main/java/com/paperless/scanner/di/AppModule.kt`
- **Lines:** `L212-L240`
- **Target:** `AppModule.provideOkHttpClient`

## Problem

OkHttpClient is built without `.cache(...)`. Even if the server emits ETag/Last-Modified, the client never benefits from 304s.

## Proposed Fix

Add `cache(Cache(context.cacheDir.resolve("http"), 50 * 1024 * 1024))`. Configure Cache-Control on read-only Retrofit calls (`@Headers("Cache-Control: max-age=300")` or via interceptor). Bypass cache on logout (clear cache).

## Acceptance Criteria

- [ ] OkHttp Cache configured (50 MB)
- [ ] List endpoints respect cache headers
- [ ] Mutations bypass cache
- [ ] Cache cleared on logout
