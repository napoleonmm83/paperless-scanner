# F-075 — Audit hardcoded inline strings inside dialogs (LabelsScreen)

**Severity:** `medium` · **Area:** `ui/i18n` · **Effort:** `S`

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/labels/LabelsScreen.kt`
- **Lines:** `L525-L570`
- **Target:** `DeleteConfirmationDialog`

## Problem

Most user-facing strings use `stringResource()` but the DeleteConfirmationDialog body composes counts + names via string concat. Risk of unlocalized fragments leaking through.

## Proposed Fix

Use parameterized `stringResource(R.string.delete_label_confirmation, count, name)`. Audit other dialogs (DocumentFilterSheet, SimplifiedSetup) for the same pattern.

## Acceptance Criteria

- [ ] No string concatenation for user-facing text in dialogs
- [ ] Parameterized stringResource used
- [ ] Lint rule (or grep test) prevents regression
