# F-078 — Icon color fallback uses hardcoded neon-yellow instead of theme token

**Severity:** `low` · **Area:** `ui/style-guide` · **Effort:** `XS`

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/labels/LabelsScreen.kt`
- **Lines:** `L139, L344`
- **Target:** `LabelsScreen`

## Problem

Fallback color `Color(0xFFE1FF8D)` mirrors the brand primary as a literal. If the brand value ever changes the fallback drifts.

## Proposed Fix

Use `MaterialTheme.colorScheme.primary` or a named `defaultLabelColor` token from the design system.
