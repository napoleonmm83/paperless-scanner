# F-004 — network_security_config permits cleartext HTTP for ALL domains

**Severity:** `high` · **Area:** `security/network` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/31

## Location

- **File:** `app/src/main/res/xml/network_security_config.xml`
- **Lines:** `L1-L8`

## Problem

Base config sets `cleartextTrafficPermitted="true"` globally. The comment mentions self-hosted Paperless without HTTPS, but this allows MITM downgrade for ANY domain. The TokenManager `acceptedHttpHosts` allowlist is only a UI warning — not enforced at the network layer.

## Proposed Fix

Set base `cleartextTrafficPermitted="false"`. Maintain cleartext only via per-`<domain>` overrides for explicitly accepted hosts. Source the allowlist from DataStore at startup and apply via a `NetworkSecurityConfig.Builder`-like mechanism, OR validate in an OkHttp `Interceptor` before connect. Log HTTP usage at WARN (no URLs).

## Acceptance Criteria

- [ ] Base cleartext disabled in network_security_config.xml
- [ ] Per-host allowlist enforced (manifest overrides or interceptor)
- [ ] HTTP requests to non-allowlisted hosts fail closed
- [ ] Tests verify both allow + deny paths
