- **Lines:** `L148-L1328`
- **Target:** `HomeViewModel`

## Problem

Single VM owns: stats, recent docs, processing tasks, untagged + deleted counts, trash retention, pending changes, active uploads, failed syncs, tag creation, suggestions, AI, network/health monitors. 1328 LOC; mocking requires 15+ deps.

## Proposed Fix

Split into composable sub-VMs (DocumentStatsViewModel, ProcessingTasksViewModel, TagManagementViewModel, ServerHealthViewModel) and have HomeViewModel become a thin coordinator. Or use `composable<Route>` + scoped sub-VMs per section.

- [ ] HomeViewModel ≤ 600 LOC
- [ ] Each sub-VM ≤ 300 LOC and unit-testable in isolation
- [ ] All HomeScreen sections still functional
