## Problem

Three high-risk classes lack any tests: ScanViewModel (908 LOC, scan flow), AppLockManager (722 LOC, biometric/PIN security), TokenManager (734 LOC, auth token + Keystore recovery). Comparison: HomeViewModel + UploadViewModel both have tests. Bugs in security/auth paths are most likely to ship undetected.

## Proposed Fix

Add `ScanViewModelTest`, `AppLockManagerTest`, `TokenManagerTest`. Use `runTest {}` + `MainDispatcherRule` for VMs; mock biometric APIs for AppLockManager; use real DataStore (in-memory file) for TokenManager + simulate AEADBadTagException recovery (commit 30305c0). Aim ≥ 80% line coverage per file.

- [ ] Three new test files exist + run in CI
- [ ] Each ≥ 80% line coverage
- [ ] AppLockManager test covers lockout, biometric, dual-SavedStateHandle paths
- [ ] TokenManager test covers Keystore corruption + recovery
