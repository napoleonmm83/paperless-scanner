# F-086 — WidgetPreferences.removeWidgetConfig uses .apply() instead of .commit()

**Severity:** `medium` · **Area:** `widget/sharedprefs` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/113

## Location

- **File:** `app/src/main/java/com/paperless/scanner/widget/WidgetPreferences.kt`
- **Lines:** `L69-L75`
- **Target:** `WidgetPreferences.removeWidgetConfig`

## Problem

Setters use `.commit()` (synchronous, per MEMORY.md mandate) but the remove path uses `.apply()`. If `onDeleted()` proceeds before the async write lands, Glance still sees the stale config.

## Proposed Fix

Change to `.commit()` and propagate the boolean success. Log failures for diagnostics.

## Acceptance Criteria

- [ ] removeWidgetConfig calls .commit()
- [ ] Returns success boolean
- [ ] Tested by removing a widget mid-render
