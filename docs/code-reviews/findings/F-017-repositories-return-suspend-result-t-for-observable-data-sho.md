# F-017 — Repositories return suspend Result<T> for observable data — should be Flow

**Severity:** `high` · **Area:** `data/repository` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/43

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/DocumentRepository.kt`
- **Lines:** `L441-L920`
- **Target:** `DocumentRepository.getDocuments / getDocument / searchDocuments / updateDocument`

## Problem

List/detail getters use `suspend fun ... Result<T>` requiring callers to manually re-fetch on changes. CLAUDE.md mandates reactive Flow / Room Flow for observable data.

## Proposed Fix

Convert read paths to `fun observeX(): Flow<...>` backed by Room Flow. Reserve `suspend` for pure-side-effect operations (login, upload). Use `stateIn(scope, SharingStarted.WhileSubscribed(5_000), initial)` to share state across collectors.

## Acceptance Criteria

- [ ] Document list + detail expose Flow
- [ ] Caching uses Room Flow not manual suspend fetch+update
- [ ] ViewModels removed manual refresh hacks
- [ ] Tests verify reactive update on DB write
