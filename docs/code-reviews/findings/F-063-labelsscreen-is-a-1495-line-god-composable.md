# F-063 — LabelsScreen is a 1495-line god-composable

**Severity:** `high` · **Area:** `refactor/god-composable` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/91

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/labels/LabelsScreen.kt`
- **Lines:** `L106-L1495`
- **Target:** `LabelsScreen`

## Problem

Single Compose file owns LabelCard, SortFilterSheet, CreateLabelSheet, DeleteDialog, ColorPickerDialog with intermingled state. Recomposition cost is large; previews + tests are brittle.

## Proposed Fix

Extract per-sub-component files (`LabelCard.kt`, `SortFilterSheet.kt`, `CreateLabelSheet.kt`, `DeleteLabelDialog.kt`, `LabelColorPicker.kt`). Hoist state to `LabelsViewModel` with SavedStateHandle. Keep `LabelsScreen` ≤ 300 LOC orchestrator.

## Acceptance Criteria

- [ ] LabelsScreen ≤ 300 LOC
- [ ] Each child composable in its own file with @Preview
- [ ] State managed by LabelsViewModel
- [ ] No regressions in label CRUD UX
