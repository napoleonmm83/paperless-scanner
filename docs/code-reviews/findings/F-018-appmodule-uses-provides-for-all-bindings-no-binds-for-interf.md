# F-018 — AppModule uses @Provides for all bindings — no @Binds for interfaces

**Severity:** `medium` · **Area:** `di/hilt` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/44

## Location

- **File:** `app/src/main/java/com/paperless/scanner/di/AppModule.kt`
- **Lines:** `L135-L260`
- **Target:** `AppModule.providePaperlessApi, provideOkHttpClient, provideGson`

## Problem

All Hilt bindings use `@Provides` returning concrete types (Retrofit `PaperlessApi`, Gson, OkHttpClient). No interface seams → tests must rebuild full HTTP stack to inject fakes.

## Proposed Fix

Extract `PaperlessApiClient` interface (or a domain-shaped façade). Use `@Binds abstract fun bindApi(impl: PaperlessApiImpl): PaperlessApiClient`. Test module overrides binding with fake. Same for OkHttp configuration where helpful.

## Acceptance Criteria

- [ ] PaperlessApi hidden behind interface, repositories depend on interface
- [ ] Test module successfully swaps in a fake implementation
- [ ] No direct concrete-class injection in production repositories
