# F-098 — Double retry layer (interceptor + worker) makes total attempts unpredictable

**Severity:** `medium` · **Area:** `architecture/retry` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/125

## Location

- **File:** `app/src/main/java/com/paperless/scanner/worker/UploadWorker.kt`
- **Lines:** `L55-L66`
- **Target:** `UploadWorker.doWork`

## Problem

RetryInterceptor (`maxRetries=3`) and UploadWorker (`Result.retry()`) both retry. A failure can therefore amount to 3 × WorkManager-retries × 3 interceptor retries before final fail, which is opaque and risks duplicate uploads.

## Proposed Fix

Pick one place: keep retry in the worker (so progress UI can reflect attempts) and turn off interceptor retries for upload endpoints (e.g. via tag). Document the retry budget per endpoint.

## Acceptance Criteria

- [ ] Single retry strategy per endpoint
- [ ] Total attempts ≤ 3
- [ ] UI shows current attempt number
- [ ] Logs distinguish layers
