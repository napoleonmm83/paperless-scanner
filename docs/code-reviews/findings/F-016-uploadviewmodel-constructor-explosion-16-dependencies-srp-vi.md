# F-016 — UploadViewModel constructor explosion (16 dependencies) — SRP violation

**Severity:** `high` · **Area:** `di/hilt` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/42

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/upload/UploadViewModel.kt`
- **Lines:** `L49-L65`
- **Target:** `UploadViewModel.<init>`

## Problem

UploadViewModel takes 16 constructor dependencies (Context, SavedStateHandle, 4 entity repositories, queue + worker manager, network + server-health monitors, analytics, suggestion orchestrator, AI usage repo, premium feature manager, token manager, dispatcher). This is a clear God-VM signal.

## Proposed Fix

Extract a `UploadCoordinator`/`PerformUploadUseCase` that owns orchestration (queue + workers + analytics + monitors). Reduce VM to 4–5 dependencies (coordinator, suggestion source, premium gate, dispatchers). Move dispatcher injection behind a `CoroutineDispatchers` bundle.

## Acceptance Criteria

- [ ] UploadViewModel constructor ≤ 5 params
- [ ] Upload orchestration moved to UseCase/Coordinator class with own tests
- [ ] All UploadScreen flows still pass UI tests
