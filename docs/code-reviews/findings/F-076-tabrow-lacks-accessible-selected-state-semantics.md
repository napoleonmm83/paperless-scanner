# F-076 — TabRow lacks accessible selected-state semantics

**Severity:** `medium` · **Area:** `ui/accessibility` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/103

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/onboarding/SimplifiedSetupScreen.kt`
- **Lines:** `L214`
- **Target:** `SimplifiedSetupScreen`

## Problem

Tab composables do not expose `Modifier.semantics { selected = ...; role = Role.Tab }`. TalkBack cannot announce which tab is current.

## Proposed Fix

Use Material 3 `Tab` correctly with `selected = ...` and an explicit `Modifier.semantics { selected = ...; role = Role.Tab }`. Add `contentDescription` if Tab uses Icon-only content.
