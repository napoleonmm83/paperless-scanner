# F-042 — HomeViewModel.tagMap is mutable shared state without thread safety

**Severity:** `high` · **Area:** `architecture/state` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/68

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/home/HomeViewModel.kt`
- **Lines:** `L255-L258`
- **Target:** `HomeViewModel.observeTagsReactively`

## Problem

`tagMap = tags.associateBy { it.id }` writes to a shared mutable var read by multiple reactive flows. No synchronization — concurrent collectors can read partial maps, causing null tag refs.

## Proposed Fix

Convert to `private val _tagMap = MutableStateFlow<Map<Int, Tag>>(emptyMap())`. Replace direct assignment with `.update {}`. Consumers collect from the StateFlow; combine where needed.

## Acceptance Criteria

- [ ] tagMap is a private StateFlow
- [ ] All updates use .update { } block syntax
- [ ] Consumers collect via Flow not direct field access
- [ ] Test verifies concurrent updates do not lose entries
