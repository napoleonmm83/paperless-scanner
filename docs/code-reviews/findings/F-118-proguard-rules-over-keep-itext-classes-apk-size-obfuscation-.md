# F-118 — ProGuard rules over-keep iText classes — APK size + obfuscation cost

**Severity:** `low` · **Area:** `tooling/proguard` · **Effort:** `XS`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/145

## Location

- **File:** `app/proguard-rules.pro`
- **Lines:** `L46-L61`
- **Target:** `iText keep rules`

## Problem

`-keep class com.itextpdf.** { *; }` keeps every iText class fully unobfuscated. The companion `-keep abstract class com.itextpdf.** { *; }` is redundant. Net effect: larger DEX, slower R8 pass.

## Proposed Fix

Replace blanket keeps with targeted `-keepclassmembers` rules for the entry points actually invoked from the app (likely PdfDocument, PdfWriter, ImageDataFactory). Run R8 with `-printseeds` / `-printusage` to identify the real surface area. Measure APK size before/after.

## Acceptance Criteria

- [ ] iText keep rules narrowed to required entry points
- [ ] R8 build succeeds without iText runtime crashes
- [ ] APK size delta documented
