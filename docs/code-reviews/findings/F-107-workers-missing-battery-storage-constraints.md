# F-107 — Workers missing battery / storage Constraints

**Severity:** `medium` · **Area:** `worker/workmanager` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/134

## Location

- **File:** `app/src/main/java/com/paperless/scanner/worker/UploadWorkManager.kt`
- **Lines:** `L25-L38`
- **Target:** `UploadWorkManager.scheduleUpload, TrashDeleteWorker scheduling`

## Problem

UploadWorker requires CONNECTED but not `requiresBatteryNotLow`. Multi-page uploads on low battery can stall mid-transfer. TrashDeleteWorker has no constraints at all.

## Proposed Fix

Add `setRequiresBatteryNotLow(true)` to upload constraints. For large uploads (>100 MB) optionally `setRequiresCharging(true)`. Add `requiresStorageNotLow(true)`. Apply CONNECTED to TrashDeleteWorker.
