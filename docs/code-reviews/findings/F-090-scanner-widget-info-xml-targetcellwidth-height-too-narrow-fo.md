# F-090 — scanner_widget_info.xml targetCellWidth/Height too narrow for supported layouts

**Severity:** `low` · **Area:** `widget/manifest` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/117

## Location

- **File:** `app/src/main/res/xml/scanner_widget_info.xml`
- **Lines:** `L9-L10`
- **Target:** `appwidget-provider`

## Problem

Metadata declares 3×1 but the widget supports 3×1, 2×2, and 3×2+ via SizeMode.Exact. System pickers can render previews and resize handles incorrectly.

## Proposed Fix

Set `targetCellWidth` / `targetCellHeight` to the smallest fully supported layout (e.g. 2×1), keep `resizeMode="horizontal|vertical"`, and refresh the preview drawable to match.

- [ ] Metadata matches smallest viable layout

- [ ] Resizing covers all designed sizes without clip
