# F-114 — MainActivity.onCreate runBlocks on tokenManager.token.first() — startup-blocking

**Severity:** `high` · **Area:** `performance/threading` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/141

## Location

- **File:** `app/src/main/java/com/paperless/scanner/MainActivity.kt`
- **Lines:** `L126-L130`
- **Target:** `MainActivity.onCreate`

## Problem

`runBlocking { tokenManager.token.first() }` blocks the UI thread during Activity creation to decide initial navigation route. The comment claims this avoids race conditions on recreation, but it directly increases Time-to-Interactive and is an Android anti-pattern.

## Proposed Fix

Set the start destination to a Splash/Loading composable. Inside Compose use `LaunchedEffect(Unit) { tokenManager.token.first() }` to load + then navigate. Or move token resolution to `Application.onCreate()` cached into an `AtomicReference<String?>` (see F-006/F-097) so MainActivity reads it without suspending.

## Acceptance Criteria

- [ ] No runBlocking in MainActivity.onCreate
- [ ] Startup token loaded asynchronously (Splash + LaunchedEffect or Application cache)
- [ ] Profiler shows reduced TTI
- [ ] No visible jank
