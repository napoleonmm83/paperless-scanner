# F-077 — LabelsScreen state held in Composable instead of ViewModel SavedStateHandle

**Severity:** `high` · **Area:** `architecture/saved-state` · **Effort:** `M`

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/labels/LabelsScreen.kt`
- **Lines:** `L106-L140`

## Problem

`searchQuery`, `showCreateSheet`, `editingLabel` are kept via `remember { mutableStateOf(...) }`. They survive recomposition but not process death or AppLock — they should be in `LabelsViewModel` SavedStateHandle (CLAUDE.md dual-sync pattern).

## Proposed Fix

Move these into LabelsViewModel as StateFlows backed by SavedStateHandle. Use the planned `rememberDualSavedState` helper (F-023) to also mirror to NavBackStackEntry where AppLock requires it.
