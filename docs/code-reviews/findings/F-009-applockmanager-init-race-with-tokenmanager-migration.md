# F-009 — AppLockManager init race with TokenManager migration

**Severity:** `medium` · **Area:** `security/applock` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/83

## Location

- **File:** `app/src/main/java/com/paperless/scanner/util/AppLockManager.kt`
- **Lines:** `L124-L168`
- **Target:** `AppLockManager.init, AppLockManager.shouldLockSync`

## Problem

`shouldLockSync()` calls `tokenManager.hasStoredCredentials()` during init. If TokenManager's plaintext→encrypted migration is still running (or Hilt initialized AppLockManager first), this can return false transiently and AppLock is bypassed even though credentials exist.

## Proposed Fix

Block until `tokenManager.isMigrationCompleted()` returns true before computing `shouldLockSync()`. Add a debug-build `require()` to fail fast on race. Document the dependency in Hilt module ordering / kdoc.

## Acceptance Criteria

- [ ] AppLockManager init waits for TokenManager migration completion
- [ ] Debug build asserts migration completed when shouldLockSync runs
- [ ] Integration test: kill app mid-migration, verify state on relaunch
