# F-056 — ScanViewModel + UploadViewModel: repeated withContext context switches inside one coroutine

**Severity:** `medium` · **Area:** `performance/threading` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/80

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/scan/ScanViewModel.kt`
- **Lines:** `L494, L504, L605, L621 (Scan); UploadViewModel L241, L334, L409, L524, L658, L704`
- **Target:** `ScanViewModel.cropPage and similar; UploadViewModel.* upload methods`

## Problem

Multiple `withContext(Dispatchers.IO)` ↔ `Dispatchers.Main` switches inside the same coroutine cost dispatcher hops without benefit.

## Proposed Fix

Establish IO context once at function entry; switch to Main only at the final state-update step. Or push state updates through `MutableStateFlow` (thread-safe, no context switch needed).

## Acceptance Criteria

- [ ] ≤ 2 context switches per coroutine
- [ ] IO boundary established at function start
- [ ] Profiler confirms reduction in dispatcher hops
