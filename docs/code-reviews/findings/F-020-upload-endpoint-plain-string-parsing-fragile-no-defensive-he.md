# F-020 — Upload endpoint plain-string parsing fragile — no defensive helper / test

**Severity:** `medium` · **Area:** `data/api` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/46

- **File:** `app/src/main/java/com/paperless/scanner/data/api/PaperlessApi.kt`
- **Lines:** `L248-L280`
- **Target:** `PaperlessApi.uploadDocument, uploadMultiPageDocument`

## Problem

Paperless-ngx returns the upload task UUID as a plain string (with optional surrounding quotes). Current parsing relies on `response.string().trim().removeSurrounding("\"")` inline at call sites. No dedicated parser, no test pinning the format.

## Proposed Fix

Add `fun parseUploadTaskId(body: ResponseBody): Result<String>` with explicit nullability + format checks. Add unit test pinned against a captured raw response. Reference CLAUDE.md API quirks section.

## Acceptance Criteria

- [ ] Dedicated parser used at all upload call sites
- [ ] Test fixture for the known plain-string response format
- [ ] Malformed body returns Failure with diagnostic message
