# F-045 — ScanViewModel does not sync pages to Navigation BackStackEntry SavedStateHandle

**Severity:** `critical` · **Area:** `architecture/saved-state` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/86

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/scan/ScanViewModel.kt`
- **Lines:** `L128-L168, L349-L385`
- **Target:** `ScanViewModel.pageUrisStateFlow, syncPagesToSavedState`

## Problem

Pages are written only to ViewModel SavedStateHandle. Per CLAUDE.md AppLock & SavedStateHandle Pattern, the Navigation BackStackEntry SavedStateHandle must also be updated, otherwise AppLock route reconstruction loses pages.

## Proposed Fix

Centralize via `updatePagesAndSync(pages)` that updates `_uiState`, ViewModel SavedStateHandle, AND Navigation BackStackEntry SavedStateHandle (passed in via screen LaunchedEffect). Provide a `rememberDualSavedState` Compose helper.

## Acceptance Criteria

- [ ] Both SavedStateHandles updated on every page mutation
- [ ] AppLock unlock test: scan, lock, unlock — pages restored
- [ ] Process Death test: same pages survive
- [ ] Tests pass
