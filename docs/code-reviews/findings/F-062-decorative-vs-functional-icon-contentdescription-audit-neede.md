# F-062 — Decorative-vs-functional Icon contentDescription audit needed across screens

**Severity:** `medium` · **Area:** `ui/accessibility` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/90

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/`
- **Lines:** `many; see DocumentsScreen L286-L412 and SettingsScreen L263, L1103, L1237 as anchors`

## Problem

Multiple Icon composables use `contentDescription = null`. Some are decorative (acceptable), others are action icons (edit / delete / filter buttons) where TalkBack must announce purpose. The mix is currently undocumented.

## Proposed Fix

Audit every `contentDescription = null` site. For functional icons use `stringResource(R.string.cd_*)`. For decorative ones add a code comment explaining intent. Add a lint rule (e.g. ktlint custom or detekt) to flag null contentDescription without an annotation.

## Acceptance Criteria

- [ ] Functional icons have meaningful contentDescription
- [ ] Decorative icons documented with `// decorative-only` comment
- [ ] Lint check enforces the rule going forward
