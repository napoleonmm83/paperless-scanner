# F-033 — Pending-change JSON built via manual buildString — fragile, untyped

**Severity:** `medium` · **Area:** `data/caching` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/59

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/DocumentRepository.kt`
- **Lines:** `L825-L845`
- **Target:** `DocumentRepository.updateDocument`

## Problem

Manual JSON construction with `buildString { append("{...}") }` and trailing-comma trimming is error-prone. No type safety; sync-time deserialization can silently drop fields.

## Proposed Fix

Define typed `PendingChangeRequest` data class mirroring `UpdateDocumentRequest`. Use injected Gson for serialize/deserialize. Reuse model types between live request + queued change.

## Acceptance Criteria

- [ ] Manual JSON construction removed
- [ ] PendingChangeRequest typed and shared with API request type
- [ ] Round-trip test asserts no field loss
