# F-096 — RetryInterceptor uses Thread.sleep() — blocks OkHttp dispatcher

**Severity:** `high` · **Area:** `performance/interceptor` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/123

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/api/RetryInterceptor.kt`
- **Lines:** `L69`
- **Target:** `RetryInterceptor.delay`

## Problem

`Thread.sleep()` blocks an OkHttp dispatcher thread for the full backoff window (up to ~10s with current parameters). Concurrent requests pile up while the pool drains.

## Proposed Fix

Lift retry to the suspendable layer (CoroutineWorker / repository helper) and remove sleep() from the interceptor chain entirely. Keep the interceptor for stateless concerns (auth refresh) only.

## Acceptance Criteria

- [ ] No Thread.sleep in interceptor code
- [ ] Retry happens at suspend boundary with `delay()`
- [ ] Concurrent throughput unaffected during retries
- [ ] Existing retry semantics preserved
