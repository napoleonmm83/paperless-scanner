# F-049 — DocumentDetailViewModel uses local mutable lists in async coroutines

**Severity:** `medium` · **Area:** `architecture/state` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/73

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/documents/DocumentDetailViewModel.kt`
- **Lines:** `L522-L523`
- **Target:** `DocumentDetailViewModel.loadAvailableUsers`

## Problem

Two `mutableListOf()` populated asynchronously, then assigned to state. Concurrent calls can interleave + duplicate entries. No thread safety.

## Proposed Fix

Combine fetches in a single suspend function returning `Pair<List<UserInfo>, List<GroupInfo>>` and update state atomically with `_uiState.update { it.copy(...) }`.

## Acceptance Criteria

- [ ] No mutableListOf created inside coroutines for shared state
- [ ] Single suspend function returns immutable result
- [ ] Atomic state update
