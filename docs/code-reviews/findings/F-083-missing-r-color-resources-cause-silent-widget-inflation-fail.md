# F-083 — Missing R.color resources cause silent widget inflation failures

**Severity:** `medium` · **Area:** `widget/style-guide` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/110

## Location

- **File:** `app/src/main/java/com/paperless/scanner/widget/ScannerWidget.kt`
- **Lines:** `L329, L368-L374`
- **Target:** `ScannerWidget.StatusContent / CompactActionButton`

## Problem

`ColorProvider(R.color.widget_text_primary)` and similar refer to color resources without a startup validation step. Missing/renamed resources surface only at runtime as Glance crashes or invisible icons.

## Proposed Fix

Verify all `R.color.widget_*` exist in `colors.xml`. Add an instrumentation test that inflates the widget and asserts no missing-resource exception. Consider extracting widget colors into a single `widget_colors.xml`.

- [ ] All widget_* color resources present
- [ ] Inflation test in CI
