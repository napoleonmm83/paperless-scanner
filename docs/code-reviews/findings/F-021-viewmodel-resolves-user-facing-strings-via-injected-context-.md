# F-021 — ViewModel resolves user-facing strings via injected Context — UI concern leaked into VM

**Severity:** `medium` · **Area:** `architecture/layering` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/47

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/upload/UploadViewModel.kt`
- **Lines:** `scattered (context.getString call sites)`
- **Target:** `UploadViewModel`

## Problem

ViewModels inject `@ApplicationContext` and call `context.getString(R.string.X)` to produce error strings. CLAUDE.md mandates: error codes from data layer, string resolution in UI layer (`stringResource`).

## Proposed Fix

Replace VM error states with sealed `UploadError` types carrying string-resource IDs. Composables resolve via `stringResource(error.messageRes)`. Reserve VM-side Context only for non-localized work (file URIs, settings).

## Acceptance Criteria

- [ ] No context.getString in VM error paths
- [ ] UploadError sealed type carries @StringRes ids
- [ ] Compose layer resolves all user-facing strings
