# F-110 — Repository tests mock DAOs — violates CLAUDE.md "don't mock the database" rule

**Severity:** `high` · **Area:** `testing/test-coverage` · **Effort:** `L`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/137

## Location

- **File:** `app/src/test/java/com/paperless/scanner/data/repository/DocumentRepositoryTest.kt`
- **Lines:** `L10-L40`
- **Target:** `DocumentRepositoryTest, AuthRepositoryTest, CorrespondentRepositoryTest, etc.`

## Problem

All repository tests use `mockk(relaxed = true)` for Room DAOs (CachedDocumentDao, CachedTagDao, PendingChangeDao, …). CLAUDE.md is explicit: integration tests must hit a real database, not mocks (rooted in a past incident where mocked tests passed but a prod migration broke). Current setup defeats integration coverage and lets DAO/migration bugs slip.

## Proposed Fix

Migrate repository tests to use an in-memory Room database via `Room.inMemoryDatabaseBuilder(...)` + the real DAO. Keep only network/API mocks. Tag pure unit tests `@SmallTest` (mocked) and integration tests `@LargeTest` (real DB) so intent is explicit. Update CLAUDE.md if the boundary needs clarification.

## Acceptance Criteria

- [ ] Repository tests run against in-memory Room DB
- [ ] DAO mocks removed from existing tests
- [ ] Tests categorized @SmallTest vs @LargeTest
- [ ] CI runs both categories
