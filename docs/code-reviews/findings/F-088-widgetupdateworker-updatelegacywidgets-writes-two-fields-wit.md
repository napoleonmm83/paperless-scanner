# F-088 — WidgetUpdateWorker.updateLegacyWidgets writes two fields with .apply()

**Severity:** `medium` · **Area:** `widget/legacy` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/115

## Location

- **File:** `app/src/main/java/com/paperless/scanner/widget/WidgetUpdateWorker.kt`
- **Lines:** `L112-L118`
- **Target:** `WidgetUpdateWorker.updateLegacyWidgets`

## Problem

Pending count + server-online flags are batched and written via `.apply()`. They are read together at render time so any partial write surfaces inconsistent state.

## Proposed Fix

Use `.commit()` and treat both fields as a single unit. Long term: migrate this state to Glance state for the modern widget so a single atomic update covers both.

- [ ] Both fields persisted before widget redraw
