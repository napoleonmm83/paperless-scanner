# F-101 — UploadWorker catch (e: Exception) hides bugs

**Severity:** `medium` · **Area:** `worker/workmanager` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/128

## Location

- **File:** `app/src/main/java/com/paperless/scanner/worker/UploadWorker.kt`
- **Lines:** `L277-L301`
- **Target:** `UploadWorker.doWork`

## Problem

Broad catch converts any throwable (OOM, NPE, IllegalState) to a "failed" result. Real bugs become user-visible upload failures with no telemetry.

## Proposed Fix

Catch IOException / network errors → `Result.retry()`. For other Throwables, log full stack trace to Crashlytics first, then `Result.failure()`. Distinguish via a `RetryDecision.from(throwable)` helper.

- [ ] Broad catch removed; typed branches added
