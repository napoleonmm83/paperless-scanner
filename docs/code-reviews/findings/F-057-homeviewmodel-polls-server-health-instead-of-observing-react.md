# F-057 — HomeViewModel polls server health instead of observing reactive monitor

**Severity:** `medium` · **Area:** `architecture/reactive` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/81

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/home/HomeViewModel.kt`
- **Lines:** `L462-L520`
- **Target:** `HomeViewModel.startServerHealthPolling`

## Problem

Polls in a `while(true) { delay; healthCheck() }` loop despite `serverHealthMonitor.isServerReachable` already being a reactive StateFlow.

## Proposed Fix

Drop the polling job. Collect `serverHealthMonitor.isServerReachable` directly. If polling is needed, do it inside the monitor — not in every consumer.
