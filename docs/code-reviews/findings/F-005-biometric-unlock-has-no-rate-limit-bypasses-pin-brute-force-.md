# F-005 — Biometric unlock has no rate limit; bypasses PIN brute-force lockout

**Severity:** `high` · **Area:** `security/applock` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/32

## Location

- **File:** `app/src/main/java/com/paperless/scanner/util/AppLockManager.kt`
- **Lines:** `L230-L250`
- **Target:** `AppLockManager.unlockWithBiometric`

## Problem

`unlockWithBiometric()` resets failed-attempt counters on success without rate limiting, and biometric succeeds during a temporary PIN lockout. An attacker who triggers biometric repeatedly (or on a device with biometric-spoof flaws) can bypass the 30-minute PIN lockout entirely.

## Proposed Fix

(1) Throttle biometric attempts (e.g., one per 5 seconds). (2) On rapid spam, escalate to the same lockout state. (3) Verify BiometricPrompt is configured with `BIOMETRIC_STRONG` and `DEVICE_CREDENTIAL` fallback. (4) Log unlock attempts with timestamp only. Document the trade-off in CLAUDE.md.

## Acceptance Criteria

- [ ] Biometric unlock throttled to N per second
- [ ] Spam triggers temporary lockout shared with PIN path
- [ ] BiometricPrompt configured with STRONG + credential fallback
- [ ] Tests verify throttle + escalation
