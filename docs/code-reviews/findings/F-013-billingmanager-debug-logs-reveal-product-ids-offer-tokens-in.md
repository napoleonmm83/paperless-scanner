# F-013 — BillingManager debug logs reveal product IDs/offer tokens in release

**Severity:** `low` · **Area:** `security/logging` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/39

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/billing/BillingManager.kt`
- **Lines:** `L131-L150, L221-L237`
- **Target:** `BillingManager.purchasesUpdatedListener, BillingManager.launchPurchaseFlow`

## Problem

Extensive `Log.d()` calls log product IDs, offer tokens, response codes. While not strictly secret, in release builds these can be captured via Logcat by other apps with READ_LOGS or via ADB on dev devices.

## Proposed Fix

Gate all `Log.d` behind `BuildConfig.DEBUG` (or use Timber with a release tree that strips DEBUG). Keep ERROR logs always-on but redact tokens. Run `proguard-rules.pro` rule to strip Log.d from release.

## Acceptance Criteria

- [ ] Log.d calls compiled out / no-op in release build
- [ ] ProGuard rule confirms Log.d stripped (verify with apkanalyzer)
- [ ] ERROR logging continues to redact sensitive payloads
