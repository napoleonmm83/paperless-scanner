# F-024 — Repositories lack stateIn / shareIn — re-collection re-queries DB

**Severity:** `low` · **Area:** `data/repository` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/50

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/TagRepository.kt`
- **Lines:** `varies`
- **Target:** `TagRepository, CorrespondentRepository, DocumentTypeRepository, etc.`

## Problem

Suspend `getAll()` getters re-execute on every call. Even where a Flow exists, no `stateIn`/`shareIn` means each collector pays the DB round-trip on recomposition.

## Proposed Fix

Convert to `fun observeAll(): StateFlow<List<T>>` shared with `stateIn(scope, SharingStarted.WhileSubscribed(5_000), emptyList())`. Document the sharing strategy.

## Acceptance Criteria

- [ ] Hot-cached entity flows use stateIn or shareIn
- [ ] Recomposition no longer triggers redundant DB queries
- [ ] Tests verify cached emission to late subscribers
