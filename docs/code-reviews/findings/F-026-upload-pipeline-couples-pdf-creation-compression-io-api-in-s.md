- **Lines:** `L73-L249, L251-L395`

## Problem

Upload functions mix multipart form construction, iText PDF creation, bitmap sampling/compression, file IO, error mapping. Each block could change independently but is fused.

## Proposed Fix

Extract `ImageProcessorService` (sampling + compression + URI→bytes), `PdfGeneratorService` (iText), and a thin `DocumentUploadRepository` that orchestrates them. Each service ≤ 100 LOC.

## Acceptance Criteria

- [ ] ImageProcessorService + PdfGeneratorService extracted with own tests
- [ ] DocumentUploadRepository orchestrates and maps results to Result<T>
- [ ] Existing upload flow tests still green
