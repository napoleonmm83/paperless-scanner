# F-058 — All ViewModels: viewModelScope coroutines lack timeouts — UI hangs on slow repos

**Severity:** `medium` · **Area:** `architecture/threading` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/82

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/`
- **Lines:** `—`
- **Target:** `All five ViewModels (~80 launch{} call sites)`

## Problem

Repository flows have no timeout. If the Flow hangs (network, deadlock), the coroutine sits forever, UI never recovers.

## Proposed Fix

Wrap loading paths in `withTimeoutOrNull(REASONABLE_TIMEOUT)` or use `Flow<T>.timeout(...)`. On timeout emit a typed error so UI can show a retry CTA.

## Acceptance Criteria

- [ ] Loading flows have timeouts (config per use case)
- [ ] Timeout produces a user-visible error + retry option
- [ ] Tests verify timeout path
