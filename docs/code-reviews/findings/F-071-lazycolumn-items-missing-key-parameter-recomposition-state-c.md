# F-071 — LazyColumn items missing key parameter — recomposition + state corruption risk

## Problem

LazyColumn around L104 declares items without explicit `key = { it.id }`. On filter changes Compose can mismatch item state (selection, scroll position).

## Proposed Fix

Add `key` to every `items()` call. Audit other screens (DocumentsScreen, LabelsScreen, TagSuggestionsSheet) for the same defect.
