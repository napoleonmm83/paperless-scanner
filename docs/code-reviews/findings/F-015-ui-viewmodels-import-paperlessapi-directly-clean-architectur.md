# F-015 — UI ViewModels import PaperlessApi directly — Clean Architecture violation

**Severity:** `critical` · **Area:** `architecture/layering` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/41

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/diagnostics/DiagnosticsViewModel.kt`
- **Lines:** `imports section`
- **Target:** `DiagnosticsViewModel, SettingsViewModel`

## Problem

Diagnostics + Settings ViewModels import `com.paperless.scanner.data.api.PaperlessApi` directly. UI must not depend on Retrofit interfaces — that breaks dependency inversion, prevents swapping implementations, and complicates testing.

## Proposed Fix

Introduce thin domain repositories (e.g. `ServerStatusRepository`, `DiagnosticsRepository`) that wrap the PaperlessApi calls and return `Result<T>`/`Flow<T>`. Inject those into the ViewModels. Remove all `data.api.*` imports from `ui/`.

## Acceptance Criteria

- [ ] No data.api.* imports under ui/
- [ ] Each removed call is wrapped in a repository method
- [ ] Repository methods return Result<T> or Flow<T>
- [ ] Tests still pass
