# F-080 — ScanScreen scan-option cards: light-only color pair (contentColor=Black)

**Severity:** `medium` · **Area:** `ui/style-guide` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/107

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/scan/ScanScreen.kt`
- **Lines:** `L546, L556`
- **Target:** `ScanScreen scan option card`

## Problem

Cards use light-blue / light-purple containers with `contentColor = Color.Black`. Dark Tech Precision Pro requires Surface `#141414` background with neon-yellow `#E1FF8D` accents — these cards stand out as off-theme.

## Proposed Fix

Use `MaterialTheme.colorScheme.surfaceVariant` background + `onSurface` content. Differentiate scan options through icon / accent stripe rather than container color.
