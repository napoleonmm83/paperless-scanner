# F-032 — Online/offline branching mixed into CRUD methods — sync coupling

**Severity:** `high` · **Area:** `architecture/caching` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/58

## Location

- **File:** `app/src/main/java/com/paperless/scanner/data/repository/DocumentRepository.kt`
- **Lines:** `L800-L877`
- **Target:** `DocumentRepository.updateDocument`

## Problem

`updateDocument()` checks `serverHealthMonitor.isServerReachable.value`, branches into either API call or pending-change insertion with manual JSON. Same pattern repeats for delete/restore. CRUD intent mixed with sync state machine.

## Proposed Fix

Extract `DocumentSyncRepository.executeOrQueue(operation, entityId, payload)`. CRUD methods delegate to it. Centralize PendingChange serialization (use Gson, see F-033) + queue management.

## Acceptance Criteria

- [ ] CRUD methods stop touching ServerHealthMonitor directly
- [ ] DocumentSyncRepository owns branching + queue
- [ ] Offline mutation returns optimistic cached result
- [ ] Sync-on-reconnect flushes queue
