# F-055 — HomeViewModel: tagMap-dependent flow can race with tag-load flow → null tag refs

**Severity:** `medium` · **Area:** `architecture/reactive` · **Effort:** `M`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/79

## Location

- **File:** `app/src/main/java/com/paperless/scanner/ui/screens/home/HomeViewModel.kt`
- **Lines:** `L272-L275`
- **Target:** `HomeViewModel.observeRecentDocumentsReactively`

## Problem

If `observeRecentDocumentsReactively` emits before `observeTagsReactively` populates tagMap, `firstTagId?.let { tagMap[it] }` returns null intermittently. Order depends on launch timing.

## Proposed Fix

Use `combine(tagRepository.observeTags(), documentRepository.observeRecentDocuments(...)) { tags, docs -> ... }` so both sources are guaranteed before mapping.

## Acceptance Criteria

- [ ] Tag-dependent flows use combine() instead of separate observers + map
- [ ] Tests verify no null tag references under varied launch order
