# F-120 — VERIFY git hooks are actually committed (CLAUDE.md vs. inspection drift)

**Severity:** `low` · **Area:** `tooling/hooks` · **Effort:** `S`
**Issue:** https://github.com/napoleonmm83/paperless-scanner/issues/147

## Location

- **File:** `.git/hooks/pre-push, .git/hooks/pre-commit`
- **Lines:** `—`

## Problem

CLAUDE.md (Lokale CI-Checks) describes a pre-push hook that auto-rebases and runs the full release CI, plus a pre-commit hook for syntax checks. A subagent inspecting the repo claimed only the standard sample hooks exist. Either (a) the hooks live under `.git/hooks/` (not in the repo, so each clone re-installs them — risky) or (b) they were lost. Worth confirming + making them install-on-clone.

## Proposed Fix

Move the hook scripts into a versioned `.githooks/` directory at the repo root. Configure `git config core.hooksPath .githooks` in a one-time bootstrap script (run on `./scripts/setup.sh`). Document the install step in README + CLAUDE.md.

## Acceptance Criteria

- [ ] Hooks live in `.githooks/` (versioned)
- [ ] Bootstrap script switches `core.hooksPath`
- [ ] README documents the setup step
- [ ] Fresh clone runs hooks after bootstrap
